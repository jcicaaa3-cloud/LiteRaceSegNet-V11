# Architecture and evaluation notes

## Model positioning

LiteRaceSegNet is positioned as a lightweight road-damage segmentation architecture, not as a generic universal segmentation model. The design target is a binary or small-class road-damage mask where the positive class can be small, irregular, low-contrast, and boundary-sensitive.

## Architecture hypothesis

The model implements three testable assumptions:

1. **High-resolution local detail is useful for small damage boundaries.**
   The H/2 detail branch keeps local structure available until fusion.

2. **Low-cost context is still necessary.**
   The H/8 context branch and LiteASPP are intended to provide road-surface context without introducing a heavy decoder.

3. **Boundary signal should be examined inside fusion, not only after prediction.**
   The boundary prediction head produces a boundary logit that can supervise boundary learning and support feature modulation before final mask prediction.

## Public diagram wording

The GitHub diagrams use `custom LiteIR backbone` rather than `MobileNetV3 backbone`. This avoids implying that MobileNetV3 is the direct main backbone. MobileNet-style efficient CNN principles are part of the design background, but the code path is implemented in `seg/core/lightweight_race.py`.

## Evaluation policy

The repository separates three types of evidence:

| Evidence type | Meaning |
|---|---|
| Reference evidence | Preliminary values already available for the lightweight model. |
| Pending final evidence | Baseline, Boundary IoU, latency, stress, and ablation values that must be generated by official scripts. |
| Claim gate | A decision layer that determines which statements are allowed after metrics are measured. |

## Device interpretation rule

CPU and GPU results are not one direct ranking. CPU results answer field/no-GPU feasibility. GPU results answer acceleration, throughput, and memory behavior. The comparison should remain inside the same device condition.

## Claim boundary

Allowed before final runs:

- The model is implemented and can pass a smoke check.
- The reference LiteRaceSegNet has 124,509 trainable parameters.
- Existing reference evidence suggests the model can produce reasonable small-validation masks.
- Baseline, latency, Boundary IoU, stress, and ablation values are pending.

Not allowed before final runs:

- Claiming state-of-the-art performance.
- Claiming superiority over SegFormer, DeepLab, or LRASPP without final comparison tables.
- Treating preliminary validation numbers as generalization proof.
- Presenting TBD values as measured results.
